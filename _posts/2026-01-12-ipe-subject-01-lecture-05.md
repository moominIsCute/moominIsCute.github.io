## 설계의 3원리 ⭐️시험 100% 출제
- **추상화**
- **구조화**
- **모듈화**

### 추상화 적용과 미적용의 차이
- **적용**: 핵심정보만 표시 (예: 고속도로 타고 4시간)
- **미적용**: 불필요한 정보까지 모두 표시 (예: 상세경로, 암호화 알고리즘, 서버IP)

### 구조화와 모듈화
- **구조화**: 거대한 시스템을 작은 기능 단위로 나눈 것
- **모듈화**: 독립적으로 작동하고 협력을 함

---

## 설계 모델링의 2가지 관점
1. **구조 모델링**: 어떻게 생겼나? (설계도)
2. **행위 모델링**: 어떻게 동작하나? (프로세스)

## 아키텍처의 3가지 특성
- **가시성**
- **간략성**
- **추상화**: 복잡한 것은 숨기기 (예: 암호화 과정 숨김)

## 아키텍처 패턴

### 1. MVC 패턴
```
Model - Controller - View
```
- 디자이너와 개발자가 독립적으로 작업 가능

### 2. 클라이언트-서버 패턴

### 3. 파이프-필터 패턴
- **예시**: 유튜브에 동영상 올릴 때 압축하고 → 썸네일 따는 과정들

### 4. 마스터-슬레이브 패턴
- **예시**: 구글 서버(본사) - 한국, 미국, 유럽 등

---

## 설계 도구

### 구조도
- **팬인/팬아웃 계산** ⭐️매년 출제!

### HIPO (Hierarchy Input Process Output)
- 기능 중심
- IPO 구조
- 유지보수 용이한 문서화 기법

### N-S 차트
- **구조**: 순차 구조, 선택 구조, 반복 구조
- **특징**:
  - 논리 중심
  - 화살표 없음
  - 단일 출입구

---

## 코드 설계

### 코드 설계의 3가지 기능
- 식별 기능
- 분류 기능
- 배열 기능

### 코드 종류
1. **순차 코드** (일련번호)
2. **블록 코드** (카테고리)
3. **그룹 분류 코드** (예: 학번 201103012)
4. **10진 코드** (도서관 분류처럼 계층적 구조)
5. **유효 숫자식 코드** ⭐️ - 제품 스펙 그대로
6. **연상 기호 코드** (예: 삼성 SM-N986N)
