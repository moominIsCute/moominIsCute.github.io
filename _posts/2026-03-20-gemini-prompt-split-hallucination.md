타임스탬프 오차는 코드로 잡았다. 그런데 AI 모델 자체에서 나오는 문제가 남았다. 프롬프트를 뜯어고치지 않으면 해결이 안 되는 종류였다.

## 문제 1 — 집중력 분산

초기 프롬프트는 하나의 템플릿으로 모든 것을 요청했다.

> "유머, 사이다, 고구마, 감동 포인트가 있으면 모두 찾아내고 각각 점수를 매겨라."

Gemini는 영상에 유머가 없어도 억지로 유머 구간을 만들어냈다. 묵직한 드라마인데 웃긴 장면을 찾으려다 보니 정작 핵심 갈등 구간이 뭉개졌다.

## 해결 — 장르별 프롬프트 분리

만능 템플릿을 두 가지로 분리했다.

**드라마 프롬프트 (`DRAMA_PROMPT`)**
- 타겟: 갈등/대립, 반전, 감정 폭발, 텐션 빌드업
- 제약: 이동 장면, 자잘한 농담 배제. 묵직한 서사에만 집중

**예능 프롬프트 (`VARIETY_PROMPT`)**
- 타겟: 티키타카, 빵 터지는 리액션, 돌발 상황
- 제약: 진지한 빌드업 구간은 버림. 타격감 있는 웃음에만 집중

파이썬에서 `--tone` 파라미터로 분기한다.

```python
if tone == "variety":
    prompt = VARIETY_PROMPT
else:
    prompt = DRAMA_PROMPT

response = client.generate_content([prompt, video_file])
```

분리 이후 드라마에선 갈등 장면이, 예능에선 리액션 구간이 정확하게 잘려 나왔다.

## 문제 2 — 시간 뭉뚱그리기 환각

프롬프트를 고쳐도 고질적인 현상이 남았다. 시작 시간을 소수점 단위로 뽑으라 해도 "12.345초"를 "12.0초"로 퉁쳐서 대답했다. 이 1초 차이 때문에 대사 첫 마디가 잘려 나갔다.

프롬프트에 명시적인 제약을 추가했다.

```
인물이 말을 꺼내기 직전이나 핵심 동작이 일어나는 찰나를 0.1초 단위로 정확하게 파악할 것.
대충 뭉뚱그린 초 단위 절대 금지.
```

강압적인 표현이 효과가 있었다. 소수점 단위 타임스탬프가 안정적으로 나오기 시작했다.

## 배운 것

AI 모델을 믿고 다 맡기는 것과, 모델의 행동 한계를 코드와 프롬프트로 통제하는 것은 다르다. 장르를 분리하고 행동 제약을 명문화했을 때 비로소 편집 자동화가 실제로 쓸 수 있는 수준이 됐다.
